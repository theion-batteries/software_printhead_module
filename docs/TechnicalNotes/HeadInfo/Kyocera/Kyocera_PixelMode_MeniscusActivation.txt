=================================================================================================
    Some information regarding  "Pixel mode"  and "meniscus activation" for some Kyocera heads 
=================================================================================================

Those featuires are supported by those quite old head types only:

Head Type   Config name

HT_KJ4      "KJ4" 
HT_K1200    "K1200"
HT_K150     "K150"
HT_K300     "K300"
HT_K600     "K600"

-------------------------------------------------------------------------------------------------
---- Pixel mode
-------------------------------------------------------------------------------------------------

This is a kind of a limited-functionality palette (doesn't allow mapping any grey level value to any pixel value).
This "mode" of "palete mapping" is specified by just one number and its meaning also depends on the selected BPP value.
Other (newer) Kyocera head types may have a full palette functionality implemented, see the corresponding TechNotes for
the particular head type.

In Meteor Config file for these head type "Pixel mode" can be set in [DefaultParameterValues], [ControllerX]
sections, e.g:

    [Controller1]
    PixelMode = 2

It is also can be set as a CPEX_PixelMode parameter via PiSetParamEx() API.

-------------------------------------------------------------------------------------------------
---- Meniscus activation
-------------------------------------------------------------------------------------------------

Sending SIG_TOGGLE_MENISCUS allows on the fly switching on and off of meniscus activation. 
Switching between pixel modes that have different drop sizes is not allowed.
